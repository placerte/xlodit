## 1. General Description

**Project Name:** xlodit

**Type:** Python package (CLI + API)

**Purpose:**

`xlodit` is a **validator/debugger for **``** workbooks produced by automated systems** (agents, scripts, pipelines).

It:

- detects formula evaluation errors
- detects type issues (e.g., numeric stored as text)
- performs minimal structural validation
- builds **traceback chains/trees** to identify upstream causes of failing cells
- outputs **deterministic, structured JSON reports** for machine consumption

The tool is:

- **agent-first** (primary consumer)
- usable in CLI and CI workflows
- read-only (no mutation of files)

## 3. Specs

### S-260407-1 — Supported File Type

- Input MUST be `.xlsx`
- Any other format MUST result in exit code `2`

### S-260407-2 — Execution Model

The audit pipeline MUST follow:

1. Load workbook (source via openpyxl, `data_only=False`)
2. Resolve backend
3. Recalculate workbook if backend supports it
4. Load recalculated workbook (`data_only=True`) when applicable
5. Run validation rules
6. Build traceback for failing formula cells
7. Assemble report

### S-260407-3 — Backends

Supported backends:

- `none`
- `libreoffice`
- `auto`

Behavior:

- `auto` → try `libreoffice`, fallback to `none`
- explicit `libreoffice` → exit code `2` if unavailable

### S-260407-4 — Backend Reporting

Report MUST include:

```
backend: {
  requested: "auto|libreoffice|none",
  used: "auto|libreoffice|none",
  recalculated: boolean
}
```

### S-260407-5 — Validation Scope

The system MUST detect:

- formula evaluation errors
- formula error datatypes
- numeric values stored as text
- minimal structural issues
- unreadable workbook

### S-260407-6 — Severity Model

Allowed severities:

- `error`
- `warning`

Rules:

- formula errors → error
- backend failure → error
- numeric-as-text → warning

### S-260407-7 — Issue Model

Each issue MUST include:

- severity
- kind
- rule\_id
- message

Optional fields (include when applicable):

- workbook (string path)
- sheet (string)
- cell (A1 notation)
- formula (string)
- value (string or number)
- backend (string)
- details (dict)

### S-260407-8 — Issue Kinds

Allowed kinds:

- workbook\_unreadable
- backend\_unavailable
- backend\_runtime\_failure
- formula\_error
- formula\_error\_datatype
- numeric\_text
- structural\_issue

### S-260407-9 — Rule IDs

Rules MUST use stable IDs:

- WB001 → unreadable workbook
- BE001 → backend unavailable
- BE002 → backend runtime failure
- F001 → formula error token
- F002 → formula error datatype
- T001 → numeric-as-text
- S001 → structural issue

### S-260407-10 — Formula Error Detection

A formula is failing if:

- evaluated value matches tokens: `#VALUE!`, `#VALEUR!`, `#REF!`, `#DIV/0!`, `#N/A`, `#N/D`, `#NAME?`, `#NOM?`, `#NUM!`, `#NOMBRE!`, `#NULL!`, or `Err:*`
- OR cell datatype is error (when observable)

### S-260407-11 — Numeric-as-Text Detection

Detect strings matching:

- regex: `^\\d+[\\.,]\\d+$`

Do NOT attempt locale normalization beyond this.

### S-260407-12 — Structural Checks (Minimal)

- workbook must load successfully
- must contain at least one worksheet

### S-260407-13 — Traceback

For `formula_error` issues:

- MUST attempt upstream dependency traceback
- MUST build a **tree structure** (not just linear)

Traceback MUST identify:

- referenced cells
- upstream failing cells
- root candidates

### S-260407-14 — Traceback Constraints

- bounded by `max_traceback_depth` (default: 10)
- MUST detect cycles
- MUST mark partial results

### S-260407-15 — Traceback Output Structure

Each issue MAY include:

```
traceback: {
  status: "complete|partial|skipped",
  cycle_detected: boolean,
  max_depth_reached: boolean,
  nodes: [TracebackNode],
  root_candidates: [
    { sheet, cell, reason }
  ]
}
```

`TracebackNode`:

```
{
  sheet,
  cell,
  value,
  children: []
}
```

### S-260407-16 — Determinism

Issue ordering MUST follow:

1. severity (error → warning)
2. sheet name (lexicographic)
3. cell (row, then column)
4. rule\_id

### S-260407-17 — Output Model

Report MUST include:

```
{
  ok: boolean,
  audit_completed: boolean,
  backend: {...},
  summary: {
    errors: int,
    warnings: int,
    total: int
  },
  issues: []
}
```

### S-260407-18 — Exit Codes

- 0 → success (no errors)
- 1 → audit completed with errors
- 2 → tool/runtime failure

### S-260407-19 — CLI Interface

Command:

```
uv run xlodit <filepath>
```

Options:

- \--backend {auto, libreoffice, none}
- \--format {terminal, json}
- \--output&#x20;
- \--max-traceback-depth&#x20;

### S-260407-20 — Output Formats

Supported:

- JSON (primary)
- terminal summary

JSON MUST be canonical and machine-readable
